# 1. Что такое Python?

**Python** — высокоуровневый язык программирования общего назначения с динамической строгой типизацией и автоматическим управлением памятью.

### Ключевые характеристики

| Характеристика | Описание |
|----------------|----------|
| **Интерпретируемый** | Код выполняется построчно интерпретатором, не требует компиляции |
| **Динамическая типизация** | Тип переменной определяется во время выполнения, а не при объявлении |
| **Строгая типизация** | Неявное преобразование типов запрещено (нельзя сложить `"2" + 2`) |
| **Автоматическая сборка мусора** | Память освобождается автоматически, без участия программиста |
| **Объектно-ориентированный** | Всё в Python — объекты (даже функции и типы) |
| **Кроссплатформенность** | Работает на Windows, Linux, macOS, Android, iOS |

### Где используется Python?

| Область | Примеры |
|---------|---------|
| **Веб-разработка** | Django, Flask, FastAPI |
| **Data Science** | Pandas, NumPy, Jupyter |
| **Машинное обучение** | TensorFlow, PyTorch, scikit-learn |
| **Автоматизация и скрипты** | Bash-скрипты, парсинг, боты |
| **Научные вычисления** | SciPy, Matplotlib |
| **Встраиваемые системы** | Raspberry Pi, микроконтроллеры |
| **Тестирование** | PyTest, Selenium |
| **DevOps** | Ansible, SaltStack, CI/CD скрипты |

**Что значит «высокоуровневый»?**

Высокоуровневый язык — это язык, который максимально абстрагирован от аппаратного обеспечения и использует понятные человеку конструкции.

| Уровень | Примеры | Особенности |
|---------|---------|-------------|
| **Низкоуровневый** | Ассемблер, машинные коды | Работа напрямую с регистрами, памятью, инструкциями CPU |
| **Среднеуровневый** | C, C++ | Есть указатели, ручное управление памятью, компиляция в машинный код |
| **Высокоуровневый** | Python, Java, JavaScript, Ruby | Автоматическое управление памятью, абстрактные структуры данных, читаемый синтаксис |

### Итоговая таблица


| Эпитет | Что означает | Простыми словами |
|--------|--------------|------------------|
| **Высокоуровневый** | Абстрагирован от железа, понятен человеку | Не нужно знать, как работает процессор и память |
| **Общего назначения** | Подходит для любых задач | Можно делать и сайты, и нейросети, и игры |
| **Динамическая типизация** | Тип переменной определяется во время выполнения | Переменная может хранить что угодно, тип не указываем |
| **Строгая типизация** | Нельзя неявно преобразовывать типы | Нельзя сложить число со строкой без явного преобразования |
| **Автоматическое управление памятью** | Память выделяется и освобождается автоматически | Не нужно вручную вызывать malloc/free |

## Изменяемые (mutable) типы данных:

Изменяемые объекты можно модифицировать после их создания - добавлять, удалять или изменять их элементы.

1. **Списки (list)** - *Упорядоченные коллекции элементов, которые можно изменять. Поддерживают индексацию, срезы и различные методы.*

   ```python
   my_list = [1, 2, 3]
   my_list.append(4)  # [1, 2, 3, 4]
   my_list[0] = 10    # [10, 2, 3, 4]
   ```

2. **Словари (dict)** - *Неупорядоченные коллекции пар "ключ-значение". Ключи должны быть уникальными и неизменяемыми.*

   ```python
   my_dict = {'a': 1, 'b': 2}
   my_dict['c'] = 3   # {'a': 1, 'b': 2, 'c': 3}
   my_dict['a'] = 10  # {'a': 10, 'b': 2, 'c': 3}

   person = {'name': 'Анна', 'age': 25, 'city': 'Москва'}

   for key in my_dict:
      print(key) #a, b

   for key in my_dict:
      print(my_dict[key]) #1, 2

   for value in my_dict.values():
      print(value) #1, 2

   for key, value in my_dict.items():
      print(f"{key}: {value}") #a: 1 , b: 2
   ```

3. **Множества (set)** - *Неупорядоченные коллекции уникальных элементов. Поддерживают математические операции над множествами.*

   ```python
   my_set = {1, 2, 3}
   my_set.add(4)      # {1, 2, 3, 4}
   my_set.remove(2)   # {1, 3, 4}
   ```

4. **Байтовые массивы (bytearray)** - *Изменяемая последовательность байтов (целые числа от 0 до 255).*

   ```python
   my_bytes = bytearray(b'hello')
   my_bytes[0] = 72   # Изменяет 'h' на 'H'
   ```

5. **Пользовательские классы** - *Объекты классов по умолчанию изменяемы - можно менять их атрибуты.*
   ```python
   class MyClass:
       def __init__(self, x):
           self.x = x
   
   obj = MyClass(5)
   obj.x = 10  # Можно изменить
   ```

## Неизменяемые (immutable) типы данных:

Неизменяемые объекты нельзя изменить после создания. Любая операция, которая "изменяет" такой объект, создаёт новый объект.

1. **Числа (int, float, complex)**

   ```python
   x = 5
   # x = 6 создаёт новый объект, старый уничтожается
   ```

2. **Строки (str)**

   ```python
   s = "hello"
   # s[0] = 'H'  # Ошибка! Нельзя изменить
   s = s + " world"  # Создаётся новая строка
   ```

3. **Кортежи (tuple)** - *Как списки, но неизменяемые. Занимают меньше памяти и быстрее списков.*

   ```python
   t = (1, 2, 3)
   # t[0] = 10  # Ошибка! Нельзя изменить
   t = (1, 2, 3, 4)  # Создаётся новый кортеж
   ```

4. **Frozenset (замороженное множество)** - *Как множество, но неизменяемое. Можно использовать как ключ словаря.*

   ```python
   fs = frozenset([1, 2, 3])
   # fs.add(4)  # Ошибка! Нельзя изменить
   ```

5. **Булевы значения (bool)**
   ```python
   flag = True
   flag = False  # Новый объект
   ```

6. **NoneType** - *Единственный объект None, представляющий отсутствие значения.*
   ```python
   x = None
   # None нельзя изменить
   ```


## Важные особенности:

1. **Копирование:**
   ```python
   # Для изменяемых типов нужно быть осторожным с присваиванием
   list1 = [1, 2, 3]
   list2 = list1  # Обе переменные ссылаются на один объект
   list2.append(4)
   print(list1)  # [1, 2, 3, 4] - изменился и list1!
   
   # Для неизменяемых типов такой проблемы нет
   s1 = "hello"
   s2 = s1  # Обе ссылаются на одну строку
   s2 = s2 + " world"  # s2 теперь ссылается на новый объект
   print(s1)  # "hello" - не изменилась
   ```

2. **Использование в качестве ключей словаря:**
   ```python
   # Можно использовать только неизменяемые типы
   d = {}
   d[(1, 2)] = "tuple key"  # OK - кортеж неизменяем
   # d[[1, 2]] = "list key"  # Ошибка! Список изменяем
   ```

3. **Производительность:**
   - Изменяемые типы эффективнее при частых модификациях
   - Неизменяемые типы безопаснее в многопоточных программах и могут использоваться как ключи в dict

Вложенные структуры:

```python
# Кортеж, содержащий список (частично изменяемый)
mixed = (1, 2, [3, 4])
# mixed[0] = 10        # Ошибка! Кортеж неизменяем

```

11. **Пример объяснения для форматирования строк:**

    *Форматирование строк* позволяет вставлять переменные и значения в текстовые строки. В Python существует несколько способов форматирования строк.

    **Старый способ с оператором `%`:**

    Старый способ использует оператор `%` для вставки значений в строку. Переменные вставляются в строку с помощью `%`, их порядок указывается после строки.

    ```python
    name = "Алиса"
    age = 25
    message = "Привет, меня зовут %s и мне %d лет." % (name, age)
    ```

    **Метод `str.format()`:**

    Метод `format()` применяется к строке и использует фигурные скобки `{}` для указания мест, куда нужно вставить значения.

    ```python
    name = "Алиса"
    age = 25
    message = "Привет, меня зовут {} и мне {} лет.".format(name, age)
    ```

    **f-строки (f-strings):**

    f-строки - это новый способ форматирования строк, доступный начиная с Python 3.6. Они используют префикс `f` перед строкой и позволяют вставлять значения переменных непосредственно внутри строки.

    ```python
    name = "Алиса"
    age = 25
    message = f"Привет, меня зовут {name} и мне {age} лет."
    ```

    Этот способ часто считается более читаемым и удобным.

    **Выравнивание и заполнение:**

    Форматирование также позволяет настраивать выравнивание и заполнение строк. Для этого используются дополнительные параметры в фигурных скобках.

    ```python
    name = "Алиса"
    age = 25
    message = f"Привет, меня зовут {name:<10} и мне {age:03} лет."
    ```

    В этом примере, `{name:<10}` означает, что переменная `name` будет выровнена по левому краю и заполнена пробелами до ширины в 10 символов, а `{age:03}` означает, что переменная `age` будет выровнена по правому краю и заполнена нулями до ширины в 3 символа.


12. **Типы строк:**

	1. **Обычные строки (`str`):** Это стандартные строки, которые мы используем. Пример: `"Привет, Мир!"`

	2. **Сырые строки (`raw string`):** Сырые строки обозначаются префиксом `r` перед строкой. В сырых строках Escape-последовательности игнорируются и интерпретируются как обычные символы.

	3. **Байтовые строки (`bytes`):** Байтовые строки обозначаются префиксом `b` перед строкой. Они представляют последовательность байтов вместо символов и могут использоваться, например, при работе с файлами.

	   ```python
	   data = b"Hello, World!"
	   ```
	   
	4. **Форматированные строки (`f-strings`):** Форматированные строки позволяют встраивать выражения и переменные непосредственно в строку с помощью префикса `f`.

	   ```python
	   name = "Алиса"

	   message = f"Привет, меня зовут {name} и мне {age} лет."
	   ```

13. ***Спецификации форматирования* для строк:**

	Это описание *спецификации форматирования* для строк в Python, используемое в методе `.format()` и f-строках. Спецификация форматирования определяет, как значения должны быть представлены в строке, включая выравнивание, ширину поля, количество знаков после запятой и другие параметры.

	Пример использования спецификации форматирования в методе f-строках:

	```python
	name = "Максим"
	age = 17
	message = f"Привет, меня зовут {name:^10} и мне {age:02d} лет."
	```
	Вывод:
	```
	Привет, меня зовут   Максим   и мне 17 лет.
	```
	В данном примере:
	- `:^10` означает выравнивание по центру в поле шириной 10 символов.
	- `:02d` означает целое число, отображаемое с ведущими нулями, чтобы всегда занимать две позиции.

	**примеры использования каждой опции спецификации форматирования:**

	- **`fill`:** Заполняющий символ для выравнивания.

	  ```python
	  name = "Alice"
	  message = "Name: {:*>10}".format(name)
	  print(message)  # Выведет "Name: *****Alice"
	  ```

	- **`align`:** Способ выравнивания внутри поля.

	  ```python
	  value = 42
	  message = "Value: {:<10}".format(value)
	  print(message)  # Выведет "Value: 42        "
	  ```

	- **`sign`:** Опции отображения знака числа.

	  ```python
	  num = -15
	  message = "Number: {:+}".format(num)
	  print(message)  # Выведет "Number: -15"
	  ```

	- **`width`:** Ширина поля, в котором будет отображено значение.

	  ```python
	  text = "Hello"
	  message = "Text: {:20}".format(text)
	  print(message)  # Выведет "Text: Hello               "
	  ```

	- **`grouping_option`:** Опция разделения групп разрядов.

	  ```python
	  amount = 1000000
	  message = "Amount: {:,}".format(amount)
	  print(message)  # Выведет "Amount: 1,000,000"
	  ```

	- **`precision`:** Количество знаков после запятой.

	  ```python
	  value = 3.141592653589793
	  message = "Value: {:.2f}".format(value)
	  print(message)  # Выведет "Value: 3.14"
	  ```

	- **`type`:** Тип форматирования значения.

	  ```python
	  num = 42
	  message = "Number: {:x}".format(num)
	  print(message)  # Выведет "Number: 2a"
	  ```

	| Оператор | Описание             | Пример              | Результат         |

# 5. ООП

## Введение: что такое ООП?

**Объектно-ориентированное программирование (ООП)** — это способ организации кода, когда программа строится из объектов, каждый из которых объединяет в себе данные (свойства, атрибуты) и поведение (методы, то есть функции, работающие с этими данными).

В отличие от процедурного стиля, где данные и функции существуют отдельно, в ООП мы создаём шаблоны (классы), а затем порождаем от них конкретные экземпляры (объекты). Это помогает:

- писать более понятный и структурированный код;
- переиспользовать код (наследование);
- защищать данные (инкапсуляция);
- легко расширять программу (полиморфизм).

## Основные понятия ООП

### 1. Класс и объект

**Класс** — это чертёж, описание, шаблон.
**Объект** — конкретный экземпляр класса, созданный по этому чертежу.

```python
# Определяем класс (чертёж)
class Cat:
    # Атрибут класса (общий для всех кошек)
    species = "млекопитающее"

    # Конструктор (инициализатор) — вызывается при создании объекта
    def __init__(self, name, age):
        self.name = name   # атрибут объекта
        self.age = age

    # Метод — функция внутри класса
    def meow(self):
        print(f"{self.name} говорит: Мяу!")

# Создаём объекты (экземпляры)
cat1 = Cat("Барсик", 3)
cat2 = Cat("Мурка", 2)

print(cat1.name)      # Барсик
print(cat2.age)       # 2
cat1.meow()           # Барсик говорит: Мяу!
print(cat1.species)   # млекопитающее
```

**Что здесь важно:**

- ```__init__``` — конструктор, который автоматически запускается при создании объекта (Cat(...)). Он инициализирует атрибуты.
- ```self``` — ссылка на конкретный объект, который мы создаём. Через self мы обращаемся к атрибутам и методам этого объекта.
- Атрибуты объекта (```self.name```, ```self.age```) живут внутри каждого объекта отдельно.
- Атрибуты класса (```species```) общие для всех экземпляров.


### 2. Методы — это функции внутри класса

Методы работают так же, как обычные функции, но у них есть доступ к объекту через ```self```.

```python
class Rectangle:
    def __init__(self, width, height):
        self.w = width
        self.h = height

    def area(self):          # метод без параметров, кроме self
        return self.w * self.h

    def scale(self, factor): # метод с параметрами
        self.w *= factor
        self.h *= factor
        return self

r = Rectangle(4, 5)
print(r.area())   # 20
r.scale(2)
print(r.area())   # 80
```

### 3. Инкапсуляция (скрытие данных)

Инкапсуляция — это объединение данных и методов внутри класса, а также **ограничение доступа** к некоторым атрибутам.

В Python нет жёстких приватных полей, но по соглашению:

- ```_атрибут``` — защищённый (не трогай без нужды)
- ```__атрибут``` — приватный (Python "искажает" имя, чтобы случайно не обратиться)

```python
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner          # публичный атрибут
        self.__balance = balance    # приватный атрибут

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount

    def get_balance(self):
        return self.__balance   # публичный метод для получения приватных данных

acc = BankAccount("Анна", 1000)
print(acc.owner)        # Анна
# print(acc.__balance)  # Ошибка! AttributeError
print(acc.get_balance()) # 1000
acc.deposit(500)
print(acc.get_balance()) # 1500
```

Зачем это нужно? Чтобы нельзя было напрямую испортить состояние объекта (например, установить отрицательный баланс).

### 4. Наследование

Наследование позволяет создать дочерний класс на основе родительского, переняв все его атрибуты и методы, и при необходимости добавить или переопределить что-то своё.

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        pass   # заглушка

class Dog(Animal):          # Dog наследует от Animal
    def speak(self):        # переопределяем метод
        return f"{self.name} говорит: Гав!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} говорит: Мяу!"

dog = Dog("Шарик")
cat = Cat("Мурзик")
print(dog.speak())   # Шарик говорит: Гав!
print(cat.speak())   # Мурзик говорит: Мяу!
```
Дочерний класс автоматически получает все методы родителя. Можно вызывать super() для доступа к родительским методам:

```python
class Child(Dog):
    def speak(self):
        # Сначала вызываем родительский метод
        parent_speak = super().speak()
        return parent_speak + " И ещё виляет хвостом!"

c = Child("Рекс")
print(c.speak())   # Рекс говорит: Гав! И ещё виляет хвостом!
```

### 5. Полиморфизм

Полиморфизм — это способность одного и того же метода работать с разными типами объектов. То есть мы можем написать код, который ожидает какой-то метод, и передавать туда объекты разных классов, главное — чтобы у них был этот метод.

```python
def make_sound(animal):
    print(animal.speak())   # мы не знаем, кто именно придёт, но знаем, что у него есть speak()

animals = [Dog("Бобик"), Cat("Снежок"), Dog("Тузик")]

for a in animals:
    make_sound(a)  # каждый объект сам знает, как говорить

# Бобик говорит: Гав!
# Снежок говорит: Мяу!
# Тузик говорит: Гав!
```
