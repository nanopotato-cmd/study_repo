## 1. Что такое Linux?

### Определение

**Linux** — это семейство Unix-подобных операционных систем с открытым исходным кодом. Строго говоря, Linux — это только **ядро** (kernel), а операционные системы, использующие это ядро, называются **дистрибутивами** (Ubuntu, Debian, CentOS, Arch и др.).

### История создания

| Год | Событие |
|-----|---------|
| 1969 | Создана ОС Unix в Bell Labs |
| 1983 | Ричард Столлман начал проект GNU (свободная альтернатива Unix) |
| 1991 | Линус Торвальдс создал ядро Linux для своего ПК |
| 1992 | Linux выпущен под лицензией GPL (свободное ПО) |

### Ключевые особенности Linux

| Особенность | Описание |
|-------------|----------|
| **Свободное ПО** | Исходный код открыт, можно изучать, изменять, распространять |
| **Многозадачность** | Несколько программ работают одновременно |
| **Многопользовательский режим** | Несколько пользователей могут работать в системе одновременно |
| **Модульность** | Ядро можно дополнять модулями без перезагрузки |
| **Стабильность** | Может работать годами без перезагрузки |
| **Безопасность** | Строгая система прав доступа, разделение привилегий |
| **Кроссплатформенность** | Работает на серверах, ПК, телефонах (Android), встраиваемых системах |

### Где используется Linux?

| Область | Примеры |
|---------|---------|
| **Серверы** | 90%+ всех серверов в интернете (включая Google, Facebook, Amazon) |
| **Суперкомпьютеры** | 100% из 500 самых мощных суперкомпьютеров |
| **Встраиваемые системы** | Роутеры, телевизоры, автомобили, банкоматы |
| **Мобильные устройства** | Android (основан на ядре Linux) |
| **Рабочие станции** | Ubuntu, Fedora, Arch (популярны среди разработчиков) |

## 2. Разновидности Linux (дистрибутивы)

### Что такое дистрибутив?

Дистрибутив — это ядро Linux + набор программ + система управления пакетами + графическая оболочка (опционально).

### Основные семейства дистрибутивов

| Семейство | Основа | Менеджер пакетов | Примеры |
|-----------|--------|------------------|---------|
| **Debian** | Debian | `apt` (`.deb`) | Ubuntu, Linux Mint, Kali Linux |
| **Red Hat** | Red Hat | `yum`/`dnf` (`.rpm`) | Fedora, CentOS, RHEL |
| **Arch** | Arch | `pacman` (`.pkg.tar.zst`) | Manjaro, EndeavourOS |
| **SUSE** | SUSE | `zypper` (`.rpm`) | openSUSE |
| **Alpine** | Alpine | `apk` | Alpine Linux (лёгкий, для контейнеров) |

### Популярные дистрибутивы и их назначение

| Дистрибутив | Назначение |
|-------------|------------|
| **Ubuntu** | Самый популярный для новичков, хорошая документация |
| **Debian** | Стабильный, для серверов |
| **Fedora** | Современный, для разработчиков (спонсируется Red Hat) |
| **CentOS / Rocky Linux** | Бесплатная альтернатива RHEL для серверов |
| **Arch Linux** | Для энтузиастов, всё настраивается вручную |
| **Linux Mint** | Простой, похож на Windows |
| **Kali Linux** | Для тестирования безопасности (пентеста) |
| **Alpine Linux** | Минималистичный (~5 МБ), для Docker-контейнеров |


## Дистрибутив Linux обычно состоит из следующих компонентов:
- загрузчик,
- ядро,
- система инициализации,
- демоны,
- командная оболочка,
- библиотеки и утилиты,
- графический сервер,
- пакетный менеджер.

### Загрузчик
Когда вы нажимаете кнопку включения компьютера, первым принимает управление прошивка BIOS или UEFI. Это низкоуровневое программное обеспечение выполняет первичную инициализацию и проверку железа компьютера и передаёт управление загрузчику операционной системы. Загрузчик, который сейчас используется повсеместно в Linux — GRUB2 (Grand Unified Bootloader второй версии). Раньше ещё были GRUB первой версии и Lilo, но встретить их в полях уже очень сложно. 

Загрузчик служит **одной цели** — запустить ядро операционной системы.

### Ядро
**Ядро** — это центральная часть операционной системы, программный код и функции управления аппаратным обеспечением и пользовательскими процессами. Ядро предоставляет пользовательским процессам сервисы и интерфейсы для взаимодействия с аппаратным обеспечением и окружающим миром с помощью механизма системных вызовов. Для прикладного программиста, пишущего текст программ, системные вызовы выглядят как обычные библиотечные функции.

Так как в задачи ядра операционной системы входят диспетчеризация и распределение ресурсов машины между потребителями, то можно выделить несколько подсистем — по типу ресурсов, которыми происходит управление:
- подсистема управления процессами;
- подсистема управления памятью;
- подсистема управления вводом-выводом;
- файловая подсистема.

**Функции управления процессами** обеспечивают многозадачность. Время работы на процессоре распределяется между многими процессами, и функции ядра аккуратно планируют работу так, чтобы никто не ушёл обиженным.

**Функции управления памятью** удовлетворяют запросы процессов на оперативную память. Операционная система создаёт для программистов удобные абстракции и память выглядит как непрерывный свободный набор адресов, готовый для использования.

**Функции ввода-вывода** позволяют процессам общаться с внешним миром. С помощью оперативной памяти реализуется файловый кэш и буферы блочных устройств для оптимизации работы с медленными дисками или сетевыми интерфейсами.

Файловая подсистема или файловый интерфейс в Linux реализован в виде абстракции **виртуальной файловой системы (англ. virtual file system, VFS)**. Различные способы форматирования дисков, сетевые файловые системы, архивы и даже настройки ядра доступны для программиста через вызов общих функций работы с файлами: open(), close() или write(). Это позволяет использовать файловые утилиты не только по основному их назначению, но и для анализа аппаратного обеспечения компьютера или изменения поведения операционной системы.

### Система инициализации
Система инициализации Linux — это первый процесс с идентификатором 1, на плечи которого ложатся задачи по созданию пользовательского окружения: монтирование файловых систем, запуск сервисов демонов и процессов. 

Если в мире загрузчиков GRUB 2 вытеснил других и используется повсеместно, то среди систем инициализации правит **systemd**. Раньше использовалась система `System V init`, которая последовательно, один за другим запускала **демоны**. В отличие от неё, systemd (system daemon) — это мощный и многофункциональный инструмент. Systemd распараллеливает запуск демонов и значительно ускоряет загрузку на многоядерных системах.

### Демон (англ. daemon) — компьютерная программа в UNIX-подобных системах, работающая в фоновом режиме без прямого взаимодействия с пользователем.

### Пакеты и пакетный менеджер
Установку всех компонентов программы автоматизируют с помощью пакета программного обеспечения — архива, содержащего все необходимые файлы, информацию о зависимостях и правила расположения файлов программы в файловой системе.

**Пакетный менеджер в Linux** - это программа, предназначенная для установки, удаления и управления пакетами. Пакетный менеджер позволяет удобно устанавливать и обновлять программы, а также обеспечивает автоматическое разрешение зависимостей между пакетами. Пакетные менеджеры ведут базу данных изменений и каждая установка или удаление пакета создаёт в такой базе данных транзакцию. Транзакция позволяет откатить изменения, если что-то пошло не так. На этом свойстве пакетных менеджеров основаны системы управления конфигурацией, такие как Ansible, Chef и Puppet.

## Что такое LVM

**LVM (с англ. logical volume manager — менеджер логических томов)** — это технология, которая, создаёт представление (абстракцию) для дисков и разделов, позволяя очень гибко работать с устройствами и разделами. 

Если мы получили в наследство набор дисков разного объёма, но хотим использовать их все как единое целое — поможет LVM. Она соберёт все диски в единый пул, из которого мы сможем создавать разделы нужного нам размера. При этом сами данные могут расползаться по нескольким физическим дискам.

А ещё мы можем более гибко работать с разделами, увеличивая и уменьшая их размер, вводя и выводя диски из виртуального пула, обходить ограничение таблицы разделов MBR на количество первичных разделов. Сплошные плюсы!

Разберём по пунктам, как работает LVM.

### Преимущества LVM

1. Гибкость. LVM позволяет гибко управлять хранилищем данных: создавайте, изменяйте размеры и удаляйте логические тома без необходимости переносить данные или перезагружать систему.
2. Сложность. LVM позволяет создавать сложные схемы хранения данных, такие как RAID-5 или RAID-6, объединение дисков, создание снимков и так далее. Это даёт возможность создавать надёжные и отказоустойчивые системы хранения данных.
3. Удобство резервного копирования. LVM упрощает процесс резервного копирования данных, так как позволяет создавать снимки (snapshot) логических томов для резервного копирования данных в реальном времени. Вы сможете создавать точки восстановления, основанные на состоянии данных тома на определённый момент времени.
4. Безопасность. LVM позволяет шифровать данные на логическом томе, обеспечивая дополнительную безопасность для данных. Это особенно важно для систем, содержащих конфиденциальную или чувствительную информацию.
5. Управление ресурсами. LVM позволяет эффективно использовать доступные ресурсы, такие как диски и пространство хранения. Так вы будете эффективнее управлять пространством хранения данных и предотвращать проблемы с заполнением дисков.
6. Миграция данных. LVM упрощает процесс миграции данных с одного физического диска на другой. Это может быть полезно, когда требуется заменить или обновить жёсткий диск без потери данных.
7. Расширяемость. LVM позволяет добавлять новые физические диски и объединять их с существующими логическими томами, чтобы легко расширять пространство хранения данных по мере необходимости.
8. Восстановление после сбоев. LVM обеспечивает восстановление данных после сбоев, так как позволяет создавать снимки логических томов, которые можно использовать для восстановления данных до определённого момента времени.


## 3. Архитектура Linux

### Уровни системы

| Уровень | Компоненты |
|---------|------------|
| **Пользовательские приложения** | браузер, VS Code, игры, текстовые редакторы |
| **Системные вызовы (syscalls)** | интерфейс между программами и ядром |
| **ЯДРО (KERNEL)** | |
| &nbsp;&nbsp;↳ Управление процессами | создание и завершение процессов, планирование CPU |
| &nbsp;&nbsp;↳ Управление памятью | выделение RAM, swap, виртуальная память |
| &nbsp;&nbsp;↳ Управление устройствами | драйверы, взаимодействие с железом |
| &nbsp;&nbsp;↳ Файловая система | чтение/запись, поддержка различных ФС |
| &nbsp;&nbsp;↳ Сеть | сетевые пакеты, сокеты, протоколы |
| &nbsp;&nbsp;↳ Безопасность | права доступа, изоляция процессов |
| **Аппаратное обеспечение** | CPU, RAM, диски, видеокарта, сеть |

### Что такое ядро (Kernel)?

Ядро — это "диспетчер" всей системы.

### Режимы работы процессора

| Режим | Что разрешено |
|-------|---------------|
| **Режим ядра (Ring 0)** | Всё: доступ к памяти, устройствам, специальные инструкции |
| **Пользовательский режим (Ring 3)** | Ограничен: только через системные вызовы |

Программы работают в пользовательском режиме. Когда им нужно что-то важное (открыть файл, отправить данные по сети), они делают **системный вызов**, и ядро выполняет операцию от их имени.



## 4. Что происходит при включении ПК?

### Полная цепочка загрузки Linux

| Этап | Что происходит | Компонент |
|------|----------------|-----------|
| **1** | Нажатие кнопки питания | Блок питания подаёт напряжение на материнскую плату |
| **2** | Процессор запускается и ищет код BIOS/UEFI по фиксированному адресу в памяти | CPU / Материнская плата |
| **3** | BIOS/UEFI выполняет POST (Power-On Self Test) — проверяет процессор, память, диски | BIOS/UEFI |
| **4** | BIOS/UEFI ищет загрузочное устройство (диск, SSD, USB) по порядку в настройках | BIOS/UEFI |
| **5** | Загрузчик (GRUB) считывается с загрузочного сектора диска | Bootloader (GRUB) |
| **6** | GRUB показывает меню выбора ОС (если настроено) и загружает ядро Linux в память | Bootloader (GRUB) |
| **7** | GRUB также загружает initramfs — начальный RAM-диск с минимальным набором драйверов | Bootloader (GRUB) |
| **8** | Ядро распаковывается, инициализируется и определяет устройства (CPU, RAM, диски) | Kernel (Ядро) |
| **9** | Ядро загружает драйверы из initramfs для доступа к дискам и файловым системам | Kernel (Ядро) |
| **10** | Ядро монтирует корневую файловую систему `/` в режиме "только чтение" | Kernel (Ядро) |
| **11** | Ядро запускает первый процесс — **PID 1** (systemd или init) | Kernel → systemd/init |
| **12** | systemd монтирует остальные файловые системы по `fstab` и запускает сервисы (сеть, логи, ssh, cron) | systemd |
| **13** | systemd запускает менеджер входа (Display Manager: GDM, LightDM, SDDM) | systemd |
| **14** | Менеджер входа показывает экран входа (логин/пароль) | Display Manager |
| **15** | После успешного входа запускается пользовательская сессия и графическая оболочка (GNOME, KDE, XFCE) | Desktop Environment |

## 5. Процессы в Linux

### Что такое процесс?

**Процесс** — это программа в момент выполнения. У каждого процесса есть:

- Уникальный идентификатор — **PID** (Process ID)
- Память (код, данные, стек)
- Открытые файлы и сетевые соединения
- Состояние (выполняется, спит, остановлен, зомби)

### Состояния процессов

| Состояние | Обозначение | Описание |
|-----------|-------------|----------|
| **Running** | `R` | Выполняется или в очереди на выполнение |
| **Sleeping** | `S` | Ждёт события (нажатие клавиши, данные с диска) |
| **Uninterruptible** | `D` | Ждёт ввода-вывода (нельзя прервать) |
| **Stopped** | `T` | Остановлен сигналом (например, Ctrl+Z) |
| **Zombie** | `Z` | Завершился, но родитель не прочитал статус |

### Иерархия процессов


| Уровень | PID | Процесс | Описание |
|---------|-----|---------|----------|
| 1 | 1 | systemd / init | Прародитель всех процессов |
| 2 | 100 | sshd | Сервер SSH (принимает подключения) |
| 3 | 200 | bash | Оболочка пользователя |
| 4 | 250 | ls | Временная команда (завершится) |
| 4 | 260 | grep | Временная команда (завершится) |
| 3 | 201 | bash | Оболочка другого пользователя |
| 2 | 101 | cron | Планировщик задач |
| 2 | 102 | nginx | Веб-сервер (главный процесс) |
| 3 | 103 | nginx | Рабочий процесс |
| 3 | 104 | nginx | Рабочий процесс |
| 2 | 105 | Xorg | Графический сервер |

- **PID 1** — systemd / init (прародитель всех процессов)
  - **PID 100** — sshd (сервер SSH)
    - **PID 200** — bash (оболочка пользователя)
      - **PID 250** — ls (временная команда)
      - **PID 260** — grep (временная команда)
    - **PID 201** — bash (оболочка другого пользователя)
  - **PID 101** — cron (планировщик задач)
  - **PID 102** — nginx (веб-сервер)
    - **PID 103** — nginx (рабочий процесс)
    - **PID 104** — nginx (рабочий процесс)
  - **PID 105** — Xorg (графический сервер)

  ### Важные факты об иерархии процессов

| Факт | Описание |
|------|----------|
| **PID 1** | Первый процесс, запускается ядром. Все остальные процессы — его потомки |
| **Дочерние процессы** | Создаются через системный вызов `fork()` |
| **Сироты** | Если родитель завершился раньше, процесс "усыновляется" PID 1 |
| **Зомби** | Процесс завершился, но родитель не прочитал его статус (`wait()`) |
| **Демоны** | Фоновые процессы, часто переживающие родителя (sshd, cron, nginx) |


## 6. Взаимодействие с железом

### Как Linux общается с устройствами?

В Linux **всё — это файл**. Устройства представлены специальными файлами в папке `/dev/`.

| Устройство | Файл в `/dev/` |
|------------|----------------|
| Первый жёсткий диск | `/dev/sda` |
| Первый раздел первого диска | `/dev/sda1` |
| Второй жёсткий диск | `/dev/sdb` |
| Терминал (ваше окно) | `/dev/tty1` |
| Случайные числа | `/dev/random`, `/dev/urandom` |
| Пустота (удалить данные) | `/dev/null` |
| Нули (источник нулей) | `/dev/zero` |

### Модули ядра (драйверы)

Драйверы в Linux — это **модули ядра**, которые можно загружать и выгружать без перезагрузки.

```bash
lsmod                   # Список загруженных модулей
modprobe module_name    # Загрузить модуль
modprobe -r module_name # Выгрузить модуль
modinfo module_name     # Информация о модуле
```

## 7. Файловая система

### Стандартная иерархия (FHS — Filesystem Hierarchy Standard)

| Папка | Содержимое |
|-------|------------|
| `/` | Корень файловой системы |
| `/bin` | Основные команды (`ls`, `cp`, `mv`) — доступны всем |
| `/sbin` | Системные команды (только для `root`) |
| `/etc` | Конфигурационные файлы |
| `/home` | Домашние папки пользователей |
| `/root` | Домашняя папка `root` |
| `/var` | Переменные данные (логи, кэш, спул) |
| `/tmp` | Временные файлы (очищаются при перезагрузке) |
| `/usr` | Пользовательские программы, библиотеки, документация |
| `/proc` | Виртуальная ФС с информацией о процессах |
| `/dev` | Файлы устройств |
| `/boot` | Файлы для загрузки (ядро, `initramfs`, GRUB) |

### `/proc` — окно в ядро

`/proc` — это не настоящие файлы, а образ памяти ядра. Чтение этих файлов даёт информацию о системе.

```bash
cat /proc/cpuinfo      # Информация о процессоре
cat /proc/meminfo      # Информация о памяти
cat /proc/version      # Версия ядра
cat /proc/uptime       # Время работы системы
cat /proc/loadavg      # Средняя нагрузка
ls /proc/1234/         # Информация о процессе с PID 1234

```

## 5. Права доступа

### Что такое права доступа?

В Linux каждый файл и папка имеют три категории пользователей и три типа прав. Это основа безопасности системы.

### Три категории пользователей

| Категория | Обозначение | Кто входит |
|-----------|-------------|------------|
| **Владелец** | `u` (user) | Пользователь, который создал файл |
| **Группа** | `g` (group) | Пользователи, входящие в группу файла |
| **Остальные** | `o` (others) | Все остальные пользователи системы |

### Три типа прав

| Право | Обозначение | Для файла | Для папки |
|-------|-------------|-----------|-----------|
| **Чтение** | `r` (read) | Просмотр содержимого | Просмотр списка файлов (`ls`) |
| **Запись** | `w` (write) | Изменение содержимого | Создание/удаление файлов внутри |
| **Исполнение** | `x` (execute) | Запуск как программы | Вход в папку (`cd`) |

---

### Формат отображения прав

Команда `ls -la` показывает права в таком формате:

```bash
-rwxr-xr-- 1 user group 1234 Mar 29 10:00 script.sh
d rwx r-x r-- 1 user group 4096 Mar 29 10:00 my_folder
│ │   │   │
│ │   │   └─── права для остальных (others)
│ │   └─────── права для группы (group)
│ └─────────── права для владельца (user)
└───────────── тип файла (- обычный, d папка, l ссылка)

```
