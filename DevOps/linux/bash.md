# Введение в Bash

**Bash** *(Bourne Again SHell)* — командный интерпретатор и язык программирования для Unix/Linux.

**Shell** - программа, которая принимает команды от пользователя и передаёт их ядру ОС

**Скрипт** - Текстовый файл с последовательностью команд Bash

**Shebang (`#!`)** - Первая строка скрипта, указывающая, какой интерпретатор использовать

## Переменные

| Тип         | Определение                                | Пример
|-------------|--------------------------------------------|-----------------------
| Локальные   | Доступны только в текущем скрипте          | name="John"
| Системные   | Переменные окружения ОС                    | $PATH, $HOME, $USER
| Специальные | Переменные Bash с особым значением         | $?, $$, $@

### Пример работы с переменными

```bash
# Присвоение (без пробелов вокруг =)
name="John"
age=25
path=/home/user

# Использование (с $)
echo $name
echo "Меня зовут ${name}"    # фигурные скобки для границ
echo "Мне ${age} лет"

# Конкатенация
full_name="${name} Doe"
list="${path}/documents"

# Только для чтения
readonly API_KEY="abc123"

# Удаление переменной
unset temp_var
```

### Системные переменные окружения

```bash
echo $HOME          # /home/user
echo $USER          # текущий пользователь
echo $PATH          # пути поиска программ
echo $PWD           # текущая директория
echo $SHELL         # текущая оболочка
echo $HOSTNAME      # имя компьютера
echo $RANDOM        # случайное число
```

## Ввод и вывод

```bash
# echo (автоматически добавляет перевод строки)
echo "текст"

# printf (нет автоматического перевода строки)
printf "Имя: %s\n" "$name"
```

### Ввод (read)

```bash
# Чтение одной переменной
echo "Как вас зовут?"
read name
echo "Привет, $name!"

# Чтение нескольких переменных
read first last age
echo "Имя: $first, Фамилия: $last, Возраст: $age"

# С тихой (скрытой) печатью
read -s password
echo "Пароль принят"

# Таймаут ввода
read -t 5 answer
```

## Перенаправление в Bash

Когда программа запускается, операционная система автоматически создаёт для неё три специальных канала связи (потока):

```bash
# Стандартные потоки
stdin  (0)  # ввод (по умолчанию — клавиатура)
stdout (1)  # вывод (по умолчанию — экран)
stderr (2)  # ошибки (по умолчанию — экран)

# Перенаправление
command > stdout.txt      # перенаправить stdout
command 2> stderr.txt     # перенаправить stderr
command &> all.txt        # перенаправить всё
command >> append.txt     # добавить в конец
```

### Как работает перенаправление

**Символ `>` — перенаправление stdout (перезапись)**

```bash
# Обычный вывод на экран
echo "Hello"           # Hello → на экран

# Перенаправляем stdout в файл
echo "Hello" > file.txt
# Теперь "Hello" попадает в file.txt, на экране ничего нет
```

**Символ `>>` — перенаправление stdout (добавление)**

```bash
echo "Первая строка" > file.txt   # создаёт файл
echo "Вторая строка" >> file.txt  # добавляет в конец
echo "Третья строка" >> file.txt  # добавляет ещё

cat file.txt
# Первая строка
# Вторая строка
# Третья строка
```

**Перенаправление stderr (`2>`)**

```bash
# Разделяем потоки
find / -name "test" > found.txt 2> errors.txt
# - found.txt: список найденных файлов
# - errors.txt: сообщения об ошибках
```

**Перенаправление обоих потоков (`&>`)**

```bash
# Перенаправить и stdout, и stderr в один файл
command &> all.txt     # современный синтаксис
command > all.txt 2>&1 # классический синтаксис (о нём ниже)

# Добавление обоих потоков
command &>> all.txt
```

**Классическое перенаправление: `2>&1`**

Этот синтаксис часто пугает новичков. Давайте разберём его по частям.
```bash
command > file.txt 2>&1
```

- `2` — stderr (поток ошибок)
- `>` — перенаправить
- `&1` — туда же, куда направлен поток 1 (stdout)

Простыми словами: *"Stderr, направься туда же, куда уже направлен stdout"*

## Pipe (|) — передача данных между командами

**Pipe (пайп, конвейер)** — это перенаправление stdout одной команды на stdin другой.

```bash
# Пример 1: подсчитать количество .txt файлов
ls -la | grep ".txt" | wc -l

# Пример 2: отобразить 10 самых больших файлов
du -sh * | sort -rh | head -10

# Пример 3: найти все процессы пользователя
ps aux | grep "$USER" | grep -v grep
```

### Что происходит под капотом:
- Команда ls -la выполняется и пишет stdout
- Bash направляет этот stdout напрямую на stdin команды grep
- grep читает эти данные как будто с клавиатуры
- Результат grep передаётся дальше wc -l

## Таблица всех операторов

| Оператор | Назначение                                | Пример                      |
|----------|-------------------------------------------|-----------------------------|
| `>`      | Перенаправить stdout (перезапись)         | `echo hi > file`            |
| `>>`     | Перенаправить stdout (добавление)         | `echo hi >> file`           |
| `2>`     | Перенаправить stderr                      | `cmd 2> errors`             |
| `2>>`    | Добавить stderr в файл                    | `cmd 2>> errors`            |
| `&>`     | Перенаправить stdout и stderr             | `cmd &> all`                |
| `&>>`    | Добавить оба потока                       | `cmd &>> all`               |
| `>&`     | Объединить потоки                         | `cmd > file 2>&1`           |
| `<`      | Взять stdin из файла                      | `sort < file`               |
| `<<`     | Here Document                             | `cat << EOF`                |
| `<<<`    | Here String                               | `grep hi <<< "hi there"`    |
| `\|`     | Pipe (stdout → stdin)                     | `cmd1 \| cmd2`              |
| `\|&`    | Pipe stdout+stderr                        | `cmd1 \|& cmd2`             |
