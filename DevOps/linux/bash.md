# Введение в Bash

**Bash** *(Bourne Again SHell)* — командный интерпретатор и язык программирования для Unix/Linux.

**Shell** - программа, которая принимает команды от пользователя и передаёт их ядру ОС

**Скрипт** - Текстовый файл с последовательностью команд Bash

**Shebang (`#!`)** - Первая строка скрипта, указывающая, какой интерпретатор использовать

## Переменные

| Тип         | Определение                                | Пример
|-------------|--------------------------------------------|-----------------------
| Локальные   | Доступны только в текущем скрипте          | name="John"
| Системные   | Переменные окружения ОС                    | $PATH, $HOME, $USER
| Специальные | Переменные Bash с особым значением         | $?, $$, $@

### Пример работы с переменными

```bash
# Присвоение (без пробелов вокруг =)
name="John"
age=25
path=/home/user

# Использование (с $)
echo $name
echo "Меня зовут ${name}"    # фигурные скобки для границ
echo "Мне ${age} лет"

# Конкатенация
full_name="${name} Doe"
list="${path}/documents"

# Только для чтения
readonly API_KEY="abc123"

# Удаление переменной
unset temp_var
```

### Системные переменные окружения

```bash
echo $HOME          # /home/user
echo $USER          # текущий пользователь
echo $PATH          # пути поиска программ
echo $PWD           # текущая директория
echo $SHELL         # текущая оболочка
echo $HOSTNAME      # имя компьютера
echo $RANDOM        # случайное число
```

## Ввод и вывод

```bash
# echo (автоматически добавляет перевод строки)
echo "текст"

# printf (нет автоматического перевода строки)
printf "Имя: %s\n" "$name"
```

### Ввод (read)

```bash
# Чтение одной переменной
echo "Как вас зовут?"
read name
echo "Привет, $name!"

# Чтение нескольких переменных
read first last age
echo "Имя: $first, Фамилия: $last, Возраст: $age"

# С тихой (скрытой) печатью
read -s password
echo "Пароль принят"

# Таймаут ввода
read -t 5 answer
```

## Перенаправление в Bash

Когда программа запускается, операционная система автоматически создаёт для неё три специальных канала связи (потока):

```bash
# Стандартные потоки
stdin  (0)  # ввод (по умолчанию — клавиатура)
stdout (1)  # вывод (по умолчанию — экран)
stderr (2)  # ошибки (по умолчанию — экран)

# Перенаправление
command > stdout.txt      # перенаправить stdout
command 2> stderr.txt     # перенаправить stderr
command &> all.txt        # перенаправить всё
command >> append.txt     # добавить в конец
```

### Как работает перенаправление

**Символ `>` — перенаправление stdout (перезапись)**

```bash
# Обычный вывод на экран
echo "Hello"           # Hello → на экран

# Перенаправляем stdout в файл
echo "Hello" > file.txt
# Теперь "Hello" попадает в file.txt, на экране ничего нет
```

**Символ `>>` — перенаправление stdout (добавление)**

```bash
echo "Первая строка" > file.txt   # создаёт файл
echo "Вторая строка" >> file.txt  # добавляет в конец
echo "Третья строка" >> file.txt  # добавляет ещё

cat file.txt
# Первая строка
# Вторая строка
# Третья строка
```

**Перенаправление stderr (`2>`)**

```bash
# Разделяем потоки
find / -name "test" > found.txt 2> errors.txt
# - found.txt: список найденных файлов
# - errors.txt: сообщения об ошибках
```

**Перенаправление обоих потоков (`&>`)**

```bash
# Перенаправить и stdout, и stderr в один файл
command &> all.txt     # современный синтаксис
command > all.txt 2>&1 # классический синтаксис (о нём ниже)

# Добавление обоих потоков
command &>> all.txt
```

**Классическое перенаправление: `2>&1`**

Этот синтаксис часто пугает новичков. Давайте разберём его по частям.
```bash
command > file.txt 2>&1
```

- `2` — stderr (поток ошибок)
- `>` — перенаправить
- `&1` — туда же, куда направлен поток 1 (stdout)

Простыми словами: *"Stderr, направься туда же, куда уже направлен stdout"*

## Pipe (|) — передача данных между командами

**Pipe (пайп, конвейер)** — это перенаправление stdout одной команды на stdin другой.

```bash
# Пример 1: подсчитать количество .txt файлов
ls -la | grep ".txt" | wc -l

# Пример 2: отобразить 10 самых больших файлов
du -sh * | sort -rh | head -10

# Пример 3: найти все процессы пользователя
ps aux | grep "$USER" | grep -v grep
```

### Что происходит под капотом:
- Команда ls -la выполняется и пишет stdout
- Bash направляет этот stdout напрямую на stdin команды grep
- grep читает эти данные как будто с клавиатуры
- Результат grep передаётся дальше wc -l

## Таблица всех операторов

| Оператор | Назначение                                | Пример                      |
|----------|-------------------------------------------|-----------------------------|
| `>`      | Перенаправить stdout (перезапись)         | `echo hi > file`            |
| `>>`     | Перенаправить stdout (добавление)         | `echo hi >> file`           |
| `2>`     | Перенаправить stderr                      | `cmd 2> errors`             |
| `2>>`    | Добавить stderr в файл                    | `cmd 2>> errors`            |
| `&>`     | Перенаправить stdout и stderr             | `cmd &> all`                |
| `&>>`    | Добавить оба потока                       | `cmd &>> all`               |
| `>&`     | Объединить потоки                         | `cmd > file 2>&1`           |
| `<`      | Взять stdin из файла                      | `sort < file`               |
| `<<`     | Here Document                             | `cat << EOF`                |
| `<<<`    | Here String                               | `grep hi <<< "hi there"`    |
| `\|`     | Pipe (stdout → stdin)                     | `cmd1 \| cmd2`              |
| `\|&`    | Pipe stdout+stderr                        | `cmd1 \|& cmd2`             |


### Золотое правило, которое нужно запомнить:

- `command > file` → перенаправить хороший вывод в файл
- `command 2> file` → перенаправить ошибки в файл
- `command &> file` → перенаправить всё в файл
- `command > file 2>&1` → классический способ перенаправить всё (работает везде)
- `command1 | command2` → передать вывод первой команды на ввод второй

# Управляющие конструкции

В Bash есть два основных типа управляющих конструкций:
1. Операторы ветвления `if-else`, `case` позволяют выполнить действие(я), в зависимости от (не) выполнения условия. Они могут быть полезны, когда нужно проверить наличие файла или переменной перед выполнением кода.
2. Циклические конструкции (циклы) `for, while, until, select` позволяют повторять действия несколько раз. Пригодятся, когда нужно обработать много файлов или выполнить однотипные действия для каждого элемента списка.

## Операторы ветвления

В Bash два оператора ветвления:
- `if-else`: для выполнения определённых команды в зависимости от условия.
- `case`: чтобы выбрать действие на основе сопоставления с шаблоном.
Оператор `if-else` работает так:
```bash
if [ условие ]; then
   # выполняемые команды, если условие истинно
else
   # выполняемые команды, если условие ложно
fi 
```

Оператор `else` может отсутствовать, и тогда `if` будет выглядеть так:
```bash
if [ условие ]; then
   # выполняемые команды, если условие истинно
fi 
```
Пример использования `if-else`:

```bash
var1=123
var2=321

if [ $var1 -eq  $var2 ]; then
    echo "$var1 равно $var2"
else
    echo "$var1 не равно $var2"
fi 
```

Как происходит выполнение:
1. Заводятся две переменные `var1` и `var2`, равные `123` и `321` соответственно.
2. Встречаем оператор `if`.
    - вычисляем условие в `[]`: значение переменной `$var1` равно (`-eq`) значению переменной `$var2`?
    - Нет, 123 не равно 321 → команда между `then` и `else` не выполнится.
    - Но выполнится команда между `else` и `fi`.
    - Распечатается строка `123 не равно 321` . Благодаря интерполяции вместо имён переменных будут подставлены их значения.


Также оператор `if` можно использовать с операторами `else` и `elif` для создания более сложных условий:
```bash
if условие1; then
   команды1
elif условие2; then
   команды2
else
   команды3
fi 
```

Для сравнения чисел используются:
- `-gt` альтернатива для `>`, проверяет, первое число больше второго?
- `-lt` альтернатива для `<`, проверяет, первое число меньше второго?
- `-ge` альтернатива для `>=`, проверяет, первое число больше или равно второму?
- `-le` альтернатива для `<=`, проверяет, первое число меньше или равно второму?
- `-eq` проверяет, первое число равно второму?
- `-ne` проверяет, первое число не равно второму?

```bash
a=100
b=10

if [ $a -gt $b ]; then
    echo "a больше b"
elif [ $a < $b ]; then
    echo "a меньше b"
else
    echo "a равно b"
fi 
```

Для сравнения массивов используются операторы сравнения строк:
- `=` проверяет, равна ли первая строка второй.
- `!=` проверяет, не равна ли первая строка второй.

```bash
array1=("a" "b" "c")
array2=("a" "b" "c")

if [ "${array1[*]}" = "${array2[*]}" ]; then
    echo "Массивы равны"
else
    echo "Массивы не равны"
fi 
```

Конструкция `${array2[*]}` используется для получения всех элементов массива `array2` в виде одной строки, разделённых пробелами. Это называется **«разворачиванием»** массива. То есть в `if` происходит сравнение двух строк `a b c` и `a b c`. Они равны, поэтому на экран выведется `Массивы равны`.

Помимо чисел, строк и массивов Bash позволяет сравнивать аттрибуты файлов.

В Bash сравнение атрибутов файлов можно выполнить с помощью оператора `test` или его эквивалентного сокращённого варианта `[ ]`. 

Для сравнения атрибутов файлов используются следующие операторы:
- `-e file` — проверяет, существует ли файл.
- `-f file` — проверяет, является ли файл обычным файлом (не каталогом или символической ссылкой).
- `-d file` — проверяет, является ли файл каталогом.
- `-s file` — проверяет, что файл не пустой (имеет ненулевой размер).
- `-r file` — проверяет, доступен ли файл для чтения.
- `-w file` — проверяет, доступен ли файл для записи.
- `-x file` — проверяет, доступен ли файл для выполнения.

Проверка существования файла в bash-скрипте сравнение будет выглядеть так:

```bash
file="example.txt"

if [ -e $file ]; then
    echo "Файл существует"
fi 
```

## Циклические конструкции

Циклические конструкции в Bash используются для выполнения повторяющихся операций или команд определённое количество раз или до тех пор, пока выполняется определённое условие. Они позволяют автоматизировать и упростить выполнение задач, которые требуют повторения одних и тех же действий.

К циклическим конструкциям относятся:
- `for` — для перебора элементов в массиве или в заданном диапазоне.
- `while` — выполняет блок кода, пока заданное условие истинно.
- `until` — выполняет блок кода, пока заданное условие ложно.

## Цикл `for`:

```bash
for переменная in список
do
    команды
done 
```

- `переменная` - переменная, которая будет использоваться для хранения текущего элемента списка;
- `список` - список элементов, по которым будет проходить цикл;
- `команды` - блок кода, который будет выполняться для каждого элемента из списка.

Цикл `for` для перебора элементов в массиве:

```bash
array=("python" "camel" "cherry")
for animal in "${array[@]}"
do
    echo $animal
done 
```

Цикл `for` будет выполняться для каждого элемента в массиве array. Каждый элемент будет сохранён в переменную `animal`, которая затем выводится на экран.

Для доступа к элементам массива в Bash есть следующие конструкции: `${array[*]}` и `${array[@]}`
Основное отличие между `${array[*]}` и `${array[@]}` в том, как они обрабатываются в контексте кавычек:
- Если `${array[*]}` внутри кавычек, все элементы массива будут рассматриваться как одна строка.
- Если `${array[@]}` внутри кавычек, каждый элемент массива будет рассматриваться как отдельная строка. Также полезно, когда элемент массива содержит пробел в значении.

Цикл `for` для перебора чисел в заданном диапазоне:

```bash
for i in {1..5}
do
    echo $i
done 
```

## Цикл while

Этот цикл выполняет блок кода, пока заданное условие истинно:
```bash
while условие
do
    команды
done 
```

- `условие` — логическое выражение, которое определяет, должен ли цикл продолжаться или нет. while (как и if) не анализирует условие, а выполняет команды, которые идут за ним и ориентируется на код выхода.
- `команды` — блок кода, который будет выполняться, пока условие истинно.

```bash
i=1
while [ $i -le 10 ]
do
    echo $i
    i=$((i+1))
done 
```

## Until
Цикл `until` позволяет выполнять команды, пока условие ложно:
```bash
until условие
do
    команды
done
```

## Select

Цикл `select` используется для создания интерактивного меню, где пользователь может выбирать один из предложенных вариантов. 
Синтаксис оператора `select` выглядит следующим образом:
```bash
select переменная in список
do
    команды
done 
```

- `переменная` — имя переменной, которая будет хранить выбранный пользователем вариант.
- `список` — список вариантов, из которых пользователь может выбирать. Варианты разделяются пробелами.
- `команды` — команды, которые будут выполняться после выбора пользователем варианта.

Пример использования оператора select:

```bash
#!/bin/bash

Text='Please enter your choice(1-3): '
options=("Gopher" "Camel" "Python" )

select choice in "${options[@]}"
do
    case $choice in
        "${options[0]}")
            echo "Ваше любимое животное Gopher"
            break
            ;;
         "${options[1]}")
            echo "Ваше любимое животное Camel"
            break
            ;;
         "${options[2]}")
           echo "Ваше любимое животное Python"
           break
            ;;
        *) echo "Вы не сказали, какое у вас любимое животное =(";;
    esac
done 
```

## Функции

В Bash функции используются для группировки и повторного использования блоков кода.
Они могут быть определены внутри скрипта или в отдельном файле, который затем может быть подключен к другим скриптам.
Вызов функции происходит по её имени:

```bash
function hello {
    echo "Привет, мир!"
}

hello 
```

Функции могут принимать аргументы, которые передаются в виде переменных. 
Пример функции, принимающей два аргумента и выводящей их сумму:
```bash
function sum {
    local a=$1
    local b=$2
    local result=$((a + b))
    echo "Сумма: $result"
}

sum 5 10 
```

Функции могут возвращать значения с помощью команды `return`. 
Вот функция, которая возвращает сумму двух чисел:

```bash
function sum {
    a=$1
    b=$2
    result=$((a + b))
    return $result
}

sum 5 10
sum_result=$?
echo "Сумма: $sum_result" 
```
Значение суммы сохраняется в переменной `sum_result` с помощью `$?`.

В Bash можно использовать оператор `&` для запуска функций в **фоновом режиме (asynchronously)**.
Если после имени функции добавить `&`, то она будет запущена в фоновом режиме, и выполнение скрипта продолжится без ожидания завершения функции:

```bash
function my_function {
    sleep 5
    echo "Функция завершена"
}

my_function &
echo "Скрипт продолжает выполнение" 

#  Скрипт продолжает выполнение
#  ... скрипт завершит выполнение, пройдёт до 5 секунд и выведется ...
#  Функция завершена
```
